# civic-formal — gov.uk/USWDS

Cultural commitment: public-service. Plain language. Accessibility above all. Trust via legibility.

## Allowed visual moves

- WCAG AAA contrast (7:1 body text minimum).
- Public-service blue (`#1d70b8` gov.uk style) or muted official accent.
- Plain-language copy ("Apply for a passport", not "Initiate application").
- Clear form layouts: 1 question per page, large hit targets.
- Government wordmark + crest top-left.
- Hairline rules between sections, no decorative borders.
- Black text on white. Ample line height (1.5+).
- Visible focus rings (3px solid yellow `#ffdd00` per gov.uk standard).

## Forbidden moves

- Animations beyond focus + click.
- Custom fonts that fail to load.
- Heavy graphics in hero.
- Marketing language ("revolutionary", "seamless", "powerful").
- Cookie modals (use bare banner).
- Auto-playing media.
- Hero images that aren't load-bearing.

## Example brands matching this register

- gov.uk, uswds (US Web Design System), canada.ca, singpass, mygov.au, irs.gov modern templates, nhs.uk

## Sample copywriting voice

- "Apply for a passport renewal."
- "You'll need: your old passport, a recent photo, £82.50."
- "If you're aged 75 or over, this is free."
- "There is a problem with this service."

Imperative. Reading age 9. No jargon. Numbers cited. Currency precise.

## Pairing

- Compatible directions: `modern-minimal` only.
- Compatible archetypes: `landing` (service start page), `table` (data lookup), `footer`.
- Avoid: ANY other register. Civic-formal is jealous of its register.

## Cultural anti-pattern

If users can't accomplish the task in <3 minutes — drift. Civic-formal optimizes for "person under stress completes form correctly".
