# enterprise-trust — Stripe/Atlassian/IBM

Cultural commitment: serious, reliable, document-grade. Strict grid. Charts > hero photos. Trust signaling via specificity.

## Allowed visual moves

- Strict 12-col grid. Equal column widths.
- Single muted brand accent + neutral grays.
- Real charts (line, bar, area) embedded in hero / proof sections.
- Customer logos in restrained mono treatment.
- Stat callouts ("$2.1B processed" / "99.99% uptime").
- Compliance badges (SOC2, HIPAA, ISO).
- Side nav for documentation.
- API code blocks in heroes (tasteful).

## Forbidden moves

- Decorative illustrations.
- Mascots.
- Animated gradients.
- "Reimagine X" / "Future of Y" copy.
- Stock photos of "team collaborating".
- Confetti / micro-rewards.
- Emoji in product UI.

## Example brands matching this register

- stripe, atlassian, ibm, mongodb, hashicorp, datadog, intercom (B2B side), kraken, coinbase (institutional), wise, revolut (business), nvidia

## Sample copywriting voice

- "Enterprise-grade infrastructure for global payments."
- "99.999% uptime, audited monthly."
- "Trusted by Fortune 500 — see customer list."
- "Compliance: SOC2 Type II, HIPAA, GDPR."

Third person. Stat-first. Past-performance-cited. No hyperbole.

## Pairing

- Compatible directions: `modern-minimal` (best), `tech-utility` (for ops/observability).
- Compatible archetypes: `dashboard`, `pricing`, `landing`, `chart`, `table`.
- Avoid: `brutalist`, `underground-zine`, `consumer-playful`.

## Cultural anti-pattern

If your hero says "Software that doesn't suck" or sounds like a startup pitch — drift. Enterprise-trust is calm, specific, citing receipts.
